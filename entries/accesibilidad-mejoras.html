1) ¿Por qué importa?

La accesibilidad beneficia a todas las personas: usuarios con lector de pantalla, personas con baja visión, daltonismo, movilidad limitada, o que simplemente navegan con teclado o desde un móvil a pleno sol. Además, mejora SEO, calidad de código y usabilidad general.

Objetivo: eliminar fricciones. Pequeñas mejoras suman un gran impacto.

2) Semántica y estructura

Usa encabezados en orden (h1 → h2 → h3), landmarks (<header>, <nav>, <main>, <footer>) y listas reales.

<main>
  <h1>Título de la página</h1>
  <section>
    <h2>Sección</h2>
    <p>Contenido...</p>
  </section>
</main>

3) Texto alternativo

  • Imágenes informativas: alt describe el propósito.
  • Decorativas: alt="" y/o role="presentation".
  • Evita texto en imágenes cuando puedas usar HTML/CSS.

4) Contraste y uso del color

Asegura contraste suficiente (texto normal ≥ 4.5:1, grande ≥ 3:1). No uses solo color para transmitir significado; añade iconos, texto o estilos adicionales.

Ejemplo: 7.2:1

Ejemplo: 2.4:1

5) Teclado y foco visible

Todo elemento interactivo debe ser usable con teclado (Tab/Shift+Tab/Enter/Espacio). Asegura un foco claramente visible.

/* Foco consistente para todo elemento focusable */
:focus-visible{
  outline: 3px solid #f59e0b; outline-offset: 2px;
  box-shadow: 0 0 0 4px rgba(245,158,11,.25);
}

Navega con Tab para ver el anillo de foco.

6) Formularios y mensajes

  • Relaciona <label> con for + id.
  • Añade aria-describedby para ayudas y errores.
  • Ofrece mensajes claros (texto y no solo color).
<label for="email">Correo</label>
<input id="email" type="email" aria-describedby="help-email error-email">
<small id="help-email">Usa tu correo de trabajo.</small>
<small id="error-email" role="alert">Formato no válido</small>

7) Componentes interactivos

  • Usa el elemento correcto: enlaces para navegación (<a>), botones para acciones (<button>).
  • No re-implementes controles nativos con div a menos que sea imprescindible.
<!-- Bien -->
<button type="button">Abrir modal</button>

<!-- Evitar -->
<div role="button" tabindex="0">Abrir modal</div>

8) Preferencias del usuario

Respeta prefers-reduced-motion y evita parpadeos agresivos. Ofrece controles para pausar animaciones.

@media (prefers-reduced-motion: reduce){
  * { animation-duration:.01ms !important; animation-iteration-count:1 !important; transition:none !important; }
}

9) Multimedia

  • Ofrece subtítulos/captions y, cuando sea posible, transcripciones.
  • Evita autoplay con sonido. Permite pausar/detener.
<video controls preload="metadata">
  <source src="/media/intro.webm" type="video/webm">
  <track kind="captions" src="/media/intro.es.vtt" srclang="es" label="Español">
</video>

Checklist rápida

  • Estructura semántica y orden de encabezados
  • Texto alternativo adecuado; decorativo con alt=""
  • Contraste: 4.5:1 (texto normal) y 3:1 (texto grande)
  • Todo operable con teclado; foco visible claro
  • Formularios con label, ayudas y errores anunciables
  • Usar elementos nativos para interacción
  • Respetar prefers-reduced-motion
  • Subtítulos/transcripciones para multimedia
