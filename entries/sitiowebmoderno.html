1) Define objetivos y alcance

¿Qué problema resuelve tu sitio? ¿Quién es tu usuario? Escribe 3–5 tareas clave que un visitante debe poder completar en menos de 3 clics. Prioriza contenido esencial y reduce ruido.

Menos es más: cada pantalla debe tener un objetivo claro y medible.

2) Arquitectura y stack

Para la mayoría de proyectos, un enfoque estático/híbrido con CDN es rápido y barato. Si necesitas contenido dinámico, usa API/Servicios gestionados (Auth, DB, Search) para no reinventar la rueda.

3) Diseño responsive y tipografía

Usa un sistema de grillas fluido y tipografía escalable con clamp(). Limita la paleta de colores y define espaciados consistentes.

/* Tipografía fluida */
h1{ font-size: clamp(1.8rem, 3.5vw, 2.6rem); }
p { font-size: clamp(1rem, 1.1vw, 1.075rem); }

/* Grilla fluida básica */
.contenedor{
  display:grid; gap:1rem;
  grid-template-columns: repeat(12, 1fr);
}
.col-span-6{ grid-column: span 6; }
@media (max-width: 900px){
  .contenedor{ grid-template-columns: 1fr; }
}

4) Performance (Core Web Vitals)

  • LCP: HTML ligero, imágenes optimizadas y reservando espacio.
  • CLS: evita saltos reservando width/height.
  • INP: minimiza JS y trabajo en main thread.
<!-- Reservar espacio para imágenes -->
<img src="/img/hero.avif" alt="Portada" width="1200" height="630" loading="eager" decoding="async">

<!-- CSS crítico pequeño inline; el resto diferido (si usas hojas externas) -->
/* aquí solo CSS; JS solo si aporta valor claro */

5) Accesibilidad

  • Semántica correcta (un h1, orden de h2/h3, landmarks).
  • Contraste suficiente y foco visible.
  • Alternativos de imagen y formularios con label.
/* Foco consistente */
:focus-visible{
  outline: 3px solid #f59e0b; outline-offset: 2px;
  box-shadow: 0 0 0 4px rgba(245,158,11,.25);
}

6) SEO técnico

  • Título único + meta descripción clara.
  • URLs limpias y canónicas.
  • Open Graph/Twitter y JSON-LD (Article/Breadcrumb/Organization).

7) Seguridad y privacidad

  • HTTPS en todo el sitio; cabeceras seguras (CSP, HSTS, X-Frame-Options).
  • Minimiza datos personales; política de privacidad clara.
# Ejemplo de cabeceras (servidor)
Content-Security-Policy: default-src 'self';
Strict-Transport-Security: max-age=31536000; includeSubDomains
X-Frame-Options: DENY

8) Deploy y CI/CD

Automatiza build, pruebas y publicación. Usa entornos de preview por PR y versiona assets con hash para cachear fuerte en CDN.

# Estructura de archivos cacheables
/assets/app.5f3a1.css
/assets/app.8d2c9.js
# Cache-Control: public, max-age=31536000, immutable

9) Analítica y métricas

Mide lo que importa: conversiones, embudos clave y Core Web Vitals de usuarios reales (RUM). Itera en ciclos cortos.

Checklist de lanzamiento

  • Objetivos definidos y contenidos priorizados
  • Grilla responsive + tipografía fluida con clamp()
  • Imágenes optimizadas con dimensiones fijas
  • Accesible: semántica, contraste y foco visible
  • SEO: título, description, OG/Twitter, canonicals y JSON-LD
  • Seguridad: HTTPS y cabeceras esenciales
  • CDN activo + assets con hash y caché largo
  • Analítica y monitoreo configurados
