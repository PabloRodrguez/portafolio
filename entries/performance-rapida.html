1) Mide primero

Define tu línea base de Core Web Vitals (LCP, INP, CLS) con Lighthouse, WebPageTest o DevTools Performance. Sin una métrica inicial, no sabrás si mejoras.

Tip: registra resultados antes/después; documenta cambios y su impacto.

2) Optimiza imágenes

  • Usa formatos modernos (WebP/AVIF) con <picture> y srcset.
  • Especifica width/height para evitar CLS.
  • loading="lazy" y decoding="async" en imágenes no críticas.
<picture>
  <source srcset="/img/foto@400.avif 400w, /img/foto@800.avif 800w" type="image/avif">
  <source srcset="/img/foto@400.webp 400w, /img/foto@800.webp 800w" type="image/webp">
  <img src="/img/foto@800.jpg" alt="Descripción" width="800" height="600"
       loading="lazy" decoding="async"
       sizes="(max-width: 600px) 100vw, 600px">
</picture>

3) CSS y JS mínimos y diferidos

Entrega CSS crítico inline y difiere el resto. Carga JS con defer o async y elimina dependencias innecesarias.

<!-- CSS crítico inline (pequeño) -->
<style>header{position:sticky;top:0;background:#fff}</style>

<!-- Cargar CSS no crítico -->
<link rel="preload" href="/assets/css/styles.css" as="style" onload="this.rel='stylesheet'">

<!-- JS diferido -->
<script src="/assets/js/app.js" defer></script>

4) Acelera el primer render

  • Evita scripts sincrónicos en <head>.
  • Usa content-visibility: auto; en listas largas.
/* Solo renderiza al entrar en viewport */
.listado{
  content-visibility: auto;
  contain-intrinsic-size: 1px 600px;
}

5) Fuentes web sin FOUT/FOIT

  • font-display: swap y subset por idioma/peso.
  • Preload del WOFF2 principal.
<link rel="preload" href="/fonts/Inter-400.woff2" as="font" type="font/woff2" crossorigin>
/* CSS */
@font-face{
  font-family: "Inter";
  src: url("/fonts/Inter-400.woff2") format("woff2");
  font-weight: 400; font-style: normal; font-display: swap;
}

6) Caching que ayuda de verdad

Versiona archivos estáticos (hash en el nombre) y usa cache-control agresivo en assets.

# Netlify (_headers) o servidor
/assets/*
  Cache-Control: public, max-age=31536000, immutable

7) Extras útiles

  • INP: usa passive: true en listeners de scroll.
  • Accesibilidad: semántica primero; menos JS innecesario.
  • Preferencias de usuario: respeta prefers-reduced-motion.
@media (prefers-reduced-motion: reduce){
  * { animation-duration:.01ms !important; animation-iteration-count:1 !important; transition:none !important; }
}

Checklist rápida

  • Imágenes WebP/AVIF + tamaños correctos
  • CSS crítico inline y diferir el resto
  • JS con defer / code splitting
  • Fuentes con font-display: swap + preload
  • Cache estáticos 1 año + nombres con hash
  • Mide LCP/INP/CLS antes y después
